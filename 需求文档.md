## Blacksky 单主题 post-level thread sequence propagation 实验

## 0. 给 vibe-coding-LLM-B 的角色说明

你是本项目的 vibe-coding-LLM-B，负责根据本需求文档实现 MF-DAG 在 Bluesky Social Dataset 2024 上的第一版完整实验流程。

当前任务不是重新设计论文方法，也不是只做 1–2 个模块的 MVP，而是在较小数据规模上实现 **完整 MF-DAG 闭环框架**：

```text
data stream
→ agent population
→ cohort construction
→ mean-field compression
→ memory retrieval
→ representative LLM decision
→ follower heterogeneous update
→ propagation transition
→ evaluation
→ memory update
```

可以先用少量 agents 和少量 steps 做调试，但最终代码结构必须支持完整方法、baseline 对比、日志保存和指标评估。

## 2. 主实验对象

主实验对象定义为：

```text
基于 threads.txt 的 post-level thread sequence propagation
```

具体含义：

每条 thread 来自 `threads.txt`：

```text
thread_root_id    root_time    post_id_1,post_id_2,post_id_3,...
```

其中第 3 列是 thread 内按时间排序的 post_id 序列。

再通过 `Blacksky.jsonl.gz` 建立：

```text
post_id → user_id / text / date / like_count / reply_count / repost_count / reply_to / quotes
```

实验任务是：

```text
给定 thread 早期可观测 post/user/cohort 状态，
使用 MF-DAG 模拟后续 thread 扩展，
并与真实后续 post sequence、user participation、cohort participation 进行比较。
```

注意：

`threads.txt` 中存在大量无法 join 到 Blacksky feed_posts 的 post_id。第一版实验不强行还原完整 thread tree，而是建模 **Blacksky 可观测 matched post sub-sequence**。未 matched 的 post 可作为结构性背景信息，例如 thread_len、unmatched_count、join_coverage，但不作为 agent-level 决策目标。

activity score 第一版定义为：

```text
activity_score =

字段至少包括：

repost_count_sum

```

### 3.4 构造 cohorts

cohort 构造必须模块化，后续可替换策略。

默认策略：

```text
hybrid_activity_behavior
```

第一版可简化为：

1. 先按 activity_score 分成 high / middle / low；
2. 再结合行为特征构造 cohort，例如：

   * 平均 like_count；
   * 平均 reply_count；
   * 平均 repost_count；
   * quote 使用倾向；
   * thread participation frequency；
3. 聚合为 5–10 个 cohorts。

如果工程上第一版不想引入复杂聚类，可先使用分位数分桶：

```text
cohort_count = 5
按 activity_score 分位数切分
```

但接口必须保留：

```yaml
cohort_strategy:
  name: hybrid_activity_behavior
  cohort_count: 5
```

每个 cohort 内选择 1 个 representative LLM-agent：

```text
representative = 该 cohort 中 activity_score 最高的 agent
```

输出：

```bash
outputs/processed/blacksky/cohorts.csv
outputs/processed/blacksky/representatives.csv
```

## 4. Propagation step 定义

第一版不自行发明周/月时间窗口，优先使用数据自身结构。

### 4.1 thread 内 sequence step

对每条 candidate thread，得到 matched post sequence：

```text
m_1, m_2, ..., m_k
```

其中每个 `m_i` 都能 join 到 Blacksky post 信息和 user_id。

如果 `k >= 2`，则构造：

```text
observed prefix: m_1
target continuation: m_2 ... m_k
```

如果 `k >= 3`，可以构造多步：

```text
step 1: observed [m_1]     → target [m_2]
step 2: observed [m_1,m_2] → target [m_3]
...
```

如果为了降低 API 成本，第一版可以先使用 one-step continuation：

```text
给定前 1 个 matched post，预测后续 matched users/cohorts 是否会参与。
```

### 4.2 全局时间顺序

所有 candidate threads 按 `root_time` 排序。

memory bank 只能使用当前 thread 之前的历史信息，避免使用未来信息。

建议数据划分：

```text
按 root_time 排序
前 70%: memory warm-up / history construction
后 30%: evaluation
```

如果实现复杂，第一版可以先：

```text
按 root_time 顺序处理所有 thread；
每处理一个 thread 后更新 memory；
评估时确保当前决策不读取未来 thread 的 memory。
```

## 5. MF-DAG 方法模块

## 5.1 Mean-Field Interaction Compression

输入：

```text
当前 observed thread prefix
当前 candidate agents
cohort assignment
historical agent behavior stats
current memory context
```

输出 cohort-level mean-field state：

```json
{
  "thread_id": "...",
  "step_id": 0,
  "cohort_id": "C0",
  "num_agents": 20,
  "observed_participation_count": 3,
  "observed_participation_rate": 0.15,
  "historical_participation_rate": 0.22,
  "activity_mean": 1.47,
  "engagement_mean": 3.8,
  "recent_growth": 1,
  "thread_len": 8,
  "matched_post_count": 3,
  "join_coverage": 0.375,
  "known_participant_users": [123, 456],
  "summary": "..."
}
```

mean-field 模块只负责压缩当前群体态势，不直接替 follower 做动作。

## 5.2 Collective Social Memory Bank

memory bank 是全局的，不是每个 cohort 单独一个库。

但 memory 内容分两层：

### Global memory

记录全局传播经验：

```text
过去 thread 的传播曲线
thread final size
early-to-late growth pattern
matched_post_count
growth speed
burst / stagnation pattern
topic/feed summary
```

### Cohort-conditioned memory

记录 cohort 相关经验：

```text
cohort 历史参与率
cohort 行为分布
cohort 对不同 thread 状态的响应
representative signal 后 follower 的实际响应
cohort-level prediction error
```

建议统一保存为 JSONL：

```bash
outputs/runs/<run_id>/memory/memory_events.jsonl
```

每条 memory event 至少包含：

```json
{
  "memory_id": "...",
  "memory_type": "global|cohort",
  "memory_subtype": "trajectory|behavior|feedback|phenomenon|calibration",
  "feed_name": "Blacksky",
  "thread_id": "...",
  "root_time": "202403182357",
  "step_id": 0,
  "cohort_id": "C0",
  "state_summary": {},
  "decision_signal": {},
  "real_next_state": {},
  "simulated_next_state": {},
  "prediction_error": {},
  "text_summary": "...",
  "created_at": "..."
}
```

检索接口至少支持：

```text
retrieve_global_recent(k)
retrieve_global_similar(current_thread_state, k)
retrieve_cohort_recent(cohort_id, k)
retrieve_cohort_similar(cohort_id, current_mean_field_state, k)
```

第一版相似度可以用结构化特征余弦相似度，不强制上向量数据库。后续可替换为 embedding retrieval。

## 5.3 Memory-Guided Decision Harness

harness 负责组织完整决策流程：

```text
current mean-field state
→ retrieve global memory
→ retrieve cohort-conditioned memory
→ build representative prompt
→ call representative LLM-agent
→ parse cohort-level decision signal
→ send signal to follower update
```

representative LLM-agent 输出必须是 JSON。

建议 schema：

```json
{
  "cohort_id": "C0",
  "participation_intensity": 0.35,
  "action_bias": {
    "participate": 0.35,
    "inactive": 0.65,
    "reply": 0.2,
    "quote": 0.1,
    "repost": 0.05,
    "like": 0.15
  },
  "expected_growth": "low|medium|high",
  "attitude_or_interest_shift": "stable|increasing|decreasing",
  "confidence": 0.72,
  "rationale": "..."
}
```

第一版主任务只需要使用：

```text
participate / inactive
```

reply / quote / repost / like 可以作为辅助行为分布字段保留。

## 5.4 Lightweight Follower Heterogeneous Update

follower agents 不调用 LLM。

每个 follower 根据：

```text
自身 profile
cohort-level decision signal
自身 activity_score
历史参与率
是否已经参与当前 thread
随机扰动
```

生成动作：

```text
participate / inactive
```

可选扩展动作：

```text
reply / repost / quote / like
```

第一版建议：

```text
primary action space = participate / inactive
auxiliary action space = reply / quote / repost / like / inactive
```

输出：

```bash
outputs/runs/<run_id>/simulation/follower_actions.jsonl
```

每条记录：

```json
{
  "thread_id": "...",
  "step_id": 0,
  "agent_id": "...",
  "user_id": 123,
  "cohort_id": "C0",
  "decision_signal_id": "...",
  "participation_probability": 0.42,
  "sampled_action": "participate",
  "random_seed": 42
}
```

## 5.5 Propagation State Transition

根据 follower actions 生成模拟的下一状态：

```text
simulated participating users
simulated participating cohorts
simulated participation count
simulated cohort distribution
simulated action distribution
```

并与真实 next state 对比。

真实 next state 来自 matched post sequence：

```text
target users
target cohorts
target matched posts
target continuation size
```

输出：

```bash
outputs/runs/<run_id>/simulation/state_transitions.jsonl
```

## 5.6 Memory Update

每个 step 或每条 thread 处理完成后，必须写回 memory bank。

写回内容包括：

```text
当前 mean-field state
retrieved memory ids
representative decision signal
follower action distribution
real next state
simulated next state
prediction error
short textual summary
```

这是 MF-DAG 闭环的核心，不能省略。

## 6. Baselines

第一版先做 baseline 对比，不做复杂消融。

### 6.1 Core: Full MF-DAG

完整方法：

```text
mean-field compression
+ global memory
+ cohort-conditioned memory
+ memory-guided decision harness
+ representative LLM decision
+ follower heterogeneous update
+ dynamic memory update
```

### 6.2 Baseline A: Full LLM-Agent

所有 agents 都调用 LLM 进行个体决策。

输入：

```text
agent profile
当前 observed thread state
可选历史行为摘要
```

输出：

```text
每个 agent 是否 participate
```

该 baseline 作为高成本参考方法。

注意：

Blacksky condition_2 下 agents 约 100–119 个，因此 full LLM-agent baseline 是可控的。

### 6.3 Baseline B: Mean-Field-LLM

使用：

```text
current mean-field state + representative LLM
```

但不使用：

```text
memory bank
memory retrieval
memory update
memory-guided harness
```

该 baseline 用于验证 memory bank 和 harness 的价值。

### 6.4 Baseline C: Rule-based Agent Baseline

每个 agent 使用规则化 prompt 或规则 profile 进行 LLM 决策。

特点：

```text
不使用 mean-field compression
不使用 collective memory
不使用 representative structure
```

可理解为弱化版 LLM-agent baseline。

第一版可用固定 prompt，让 LLM 根据规则化字段输出 participate / inactive。

## 7. Evaluation Metrics

### 7.1 Efficiency Metrics

```text
LLM calls
token input
token output
total token cost
runtime
average runtime per thread
LLM call reduction ratio
```

核心对比：

```text
Full LLM-Agent vs Full MF-DAG
```

理论上：

```text
Full LLM-Agent calls ≈ num_agents × num_steps
MF-DAG calls ≈ num_cohorts × num_steps
```

### 7.2 Thread Propagation Metrics

针对 matched post sub-sequence：

```text
final matched participation size MAE / RMSE
continuation size error
cumulative matched post curve MAE
growth direction accuracy
early-to-late expansion error
```

### 7.3 User / Cohort Participation Metrics

```text
user-level Precision / Recall / F1
next-user Recall@K
cohort participation distribution JS distance
cohort participation distribution KL distance
active cohort coverage
participant diversity
```

如果 user-level prediction 太稀疏，第一版重点看 cohort-level 指标。

### 7.4 Behavior Distribution Metrics

如果辅助 action type 实现完成，计算：

```text
reply / repost / quote / like / inactive 分布误差
cohort-level action distribution distance
JS distance
```

如果第一版只完成 participate / inactive，则该部分作为扩展指标保留。

### 7.5 Heterogeneity Metrics

用于判断 follower 是否退化为简单复制 representative signal：

```text
follower action entropy
inter-cohort participation difference
intra-cohort participation variance
representative signal vs actual follower actions divergence
```

## 13. 当前不做的内容

第一版暂不要求：

```text
复杂消融实验
全量 all-feeds merged 主实验
向量数据库 memory retrieval
严格 post-level reply/repost/quote tree reconstruction
大规模 1000+ agents
复杂图神经网络或训练模型
```

这些可以作为后续扩展。

## 14. 关键注意事项

1. 不要把本实验写成只实现 mean-field 或 memory bank 的孤立 demo。
   第一版规模可以小，但框架必须完整。

2. 不要强行使用 News。
   News user 太少，不适合作为第一版主实验。

3. 不要强行凑满 200 agents。
   Blacksky condition_2 下约 119 agents 已经足够第一版完整实验与 full LLM-agent baseline。

4. 不要把 `threads.txt` 第 3 列误认为 author_id。
   第 3 列是 post_id sequence。

5. 不要把 unmatched posts 当成有 user/text 的 agent 决策对象。
   第一版只对 join 到 Blacksky feed_posts 的 matched posts 构造 user-agent 传播目标。

6. memory bank 必须动态更新。
   它不是普通日志，而是参与后续决策的 collective social memory。

7. 所有模块都要保留可插拔接口。
   后续需要替换 cohort strategy、memory retrieval、follower update、baseline 和 feed/topic。
